## What this example demonstrates

- Ambiguous match (equal scores, gap < 3) surfaces both candidates to the user rather than guessing or silently skipping.
- User resolves ambiguity in one natural-language reply — skill resumes without re-running Steps 0–4.
- "show plan in detail" flow for stage transitions: all transitions listed by name before example recap.
- Completed ambiguous matches are logged in the sync doc as "user-confirmed" for audit purposes.
- Safety rail held: stage edit only for the high-confidence signal; recap-only for Series B (no stage signal).
