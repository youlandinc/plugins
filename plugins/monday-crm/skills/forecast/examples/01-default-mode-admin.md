## What this example demonstrates

- Forecast period resolved from natural-language ("Q2") before any board read.
- Missing `Forecast Category` column → ask before `create_column` (admin-only schema edit, confirmed).
- Bucket logic combines explicit category + probability/stage fallback.
- Idempotency check before `create_dashboard` (no duplicate).
- Chat mirrors the dashboard summary + URL on the final line.
- Safety rail held: no stage or amount edits, no deletes.
- Hygiene gap count surfaced but routed to `data-cleanup` rather than acted on inline.
